# 知道管理·林下管理完整产品设计方案

> 版本：1.0  
> 产品归属：知道管理·林下系列  
> 核心主张：知其变，守其常，行其简

## 1. 项目总述

### 1.1 品牌架构

```text
知道管理
└── 林下系列
    ├── 林下问答：帮助用户理解问题
    └── 林下管理：帮助用户持续解决问题
```

“知道管理”是总品牌与方法论品牌；“林下”是面向用户的陪伴型产品系列。“林下管理”沿用“林下问答”同款品牌标识、视觉语言、账户体系与经用户授权的必要记忆。

两个产品的分工如下：

| 产品 | 核心任务 | 典型问题 |
| --- | --- | --- |
| 林下问答 | 理解、启发与讨论 | 这件事应该怎么看？ |
| 林下管理 | 诊断、推演、行动与复盘 | 现在最值得做什么？ |

用户可以将“林下问答”中的某次讨论一键转为“林下管理”的正式议题，无需重新描述背景。

### 1.2 产品定位

> 以《周易》和《焦氏易林》的变化智慧理解管理情境，以主动推理分析不确定性，陪管理者一步一步把事情做好。

林下管理不是随机起卦、命理预测、古籍自动解读器或传统管理驾驶舱，而是一个以现实证据形成判断、以经典智慧解释变化、以现代管理连接行动、以真实反馈持续校准的陪伴型管理决策系统。

系统持续帮助用户回答四个问题：

1. 现在到底发生了什么？
2. 如果保持现状，事情可能走向哪里？
3. 我真正想守住和实现什么？
4. 当前最值得采取的一个行动是什么？

## 2. 品牌理念与体验原则

### 2.1 变易、不易、简易

- **变易**：环境、组织和人的状态持续变化，系统必须随证据更新判断。
- **不易**：变化中仍需守住目标、原则与关键约束。
- **简易**：再复杂的问题，也要转化为当下可执行、可验证的一步。

品牌主张：

> 知其变，守其常，行其简。

产品主张：

> 看清此刻，走好下一步。

### 2.2 四项品牌调性

#### 极简

- 一句话进入；
- 一次只问一个关键问题；
- 默认只突出一个判断；
- 默认只推荐一个行动；
- 一步只改变一个核心变量。

#### 精准

- 区分事实、推断、经典启示和管理建议；
- 结论必须有证据、置信度和适用前提；
- 建议必须指向责任人、时间、动作与验证信号。

#### 贴心

- 不制造焦虑，不轻易归咎；
- 风险提醒必须同时给出缓冲办法；
- 用户说不清时，系统负责帮助整理；
- 允许暂缓、回退和改变目标。

#### 陪伴

- 记得用户为什么出发；
- 记得此前的判断、目标和行动；
- 在合适的时间回来验证；
- 根据真实结果调整路径；
- 帮助用户形成自己的管理能力，而非制造依赖。

## 3. 总体理论架构

```text
现实事实与用户表达
          ↓
主动推理诊断引擎
          ↓
现状信念—惯性推演—目标偏好
          ↓
《周易》状态解释引擎
          ↓
《焦氏易林》变化情境引擎
          ↓
现代管理理论、方法与策略库
          ↓
一步一爻路径优化
          ↓
现实行动—观察—验证—复盘
          └────────→ 更新模型
```

### 3.1 主动推理

主动推理负责：

- 从不完整信息中推断当前状态；
- 表达判断的不确定性；
- 推演无新增干预及不同策略下的未来状态；
- 识别用户偏好、目标与代价边界；
- 比较候选行动的目标价值、信息价值、成本与风险；
- 根据真实结果更新状态、路径与模型。

### 3.2 《周易》

- 卦辞：解释整体管理状态；
- 爻辞：解释所处阶段、位置和局部处境；
- 彖、象、系辞等传文：提供时、位、中、正、应、承乘、比、变通等结构性解释原则。

### 3.3 《焦氏易林》

《焦氏易林》形成 `64 × 64 = 4096` 种“由某卦之某卦”的变化情境，其中包含不变、单爻变和多爻变。

它用于解释：

- 变化为什么发生；
- 哪些条件有利或不利；
- 哪些信号意味着路径改善或风险上升；
- 变化过程中适合采取或避免什么行动。

经典负责提供情境类比和结构启示，不负责制造统计概率，也不替代现实证据与专业判断。

### 3.4 现代管理知识库

负责完成以下转译：

```text
经典启示
→ 现代管理理论
→ 诊断方法
→ 总体策略
→ 具体行动
→ 验证信号
```

## 4. 核心状态体系

### 4.1 现状卦

现状卦是根据当前证据形成的状态信念：

```math
Q(s_t)=P(s_t\mid o_{1:t})
```

它回答：在现有证据下，我们更可能身处什么管理情境？

### 4.2 惯性卦

惯性卦是假设不增加新的管理干预，当前结构自然演进后的概率状态：

```math
Q(s_{t+T})=(B^{(0)})^TQ(s_t)
```

它回答：如果维持现有做法，事情最可能走向哪里？

### 4.3 目标卦

目标卦是由用户确认的偏好状态，回答：用户真正希望实现什么、守住什么，并且不愿承担什么代价？

### 4.4 之卦

之卦是执行一次或一组变爻后形成的新状态。林下管理的执行路径坚持相邻状态一步只改变一爻：

```math
d_H(B_t,B_{t+1})=1
```

## 5. 六爻管理模型

| 爻位 | 管理层级 | 核心内容 |
| --- | --- | --- |
| 初爻 | 基础事实 | 资源、现金、数据、基本条件 |
| 二爻 | 执行流程 | 分工、流程、产品、交付 |
| 三爻 | 协同关系 | 部门、角色、利益、边界 |
| 四爻 | 组织机制 | 权责、制度、结构、授权 |
| 五爻 | 战略领导 | 目标、取舍、核心决策 |
| 上爻 | 文化环境 | 文化惯性、发展周期、外部趋势 |

每个爻位先使用五级离散状态：

```math
x_i\in\{-2,-1,0,+1,+2\}
```

分别表示明显受阻、偏弱、未明或平衡、有效、过强且可能反噬。阴阳用于表达结构与关系，不直接等于好坏或吉凶。

## 6. 现状诊断

### 6.1 证据来源

1. 用户的自然语言描述；
2. 对关键鉴别问题的简短回答；
3. 用户自愿提供的经营与组织数据；
4. 会议纪要、报表和行动记录；
5. 此前的判断、行动与真实结果；
6. 用户授权使用的组织历史信息。

### 6.2 极低输入门槛

用户可以输入一句话、几个关键词、一段语音、一份会议纪要、一张报表，甚至只是“乱、急、累、卡住了”。系统通过渐进式追问补齐必要信息，不要求先填写长问卷。

### 6.3 诊断流程

```text
原始表达
→ 提取事实与主观判断
→ 生成多个原因假设
→ 识别关键不确定性
→ 只问一个鉴别问题
→ 更新状态概率
→ 形成候选现状卦
```

输出以概率和依据呈现，不把候选卦包装成唯一真相：

```text
当前更支持：井　　　　　　　58%
其次可能：困　　　　　　　　29%
信息不足：13%

主要依据：
能力和资源仍然存在，
问题更集中于取用和授权机制。
```

## 7. 惯性推演

推演前必须公开假设，例如：

```text
以下推演假设：
• 负责人不变；
• 预算不增加；
• 审批机制不调整；
• 外部环境没有重大突变。
```

随后展示概率分布：

```text
维持现有做法三个月：

涣 · 进一步分散　　　　　　49%
困 · 继续受阻　　　　　　　34%
解 · 局部自行缓解　　　　　17%
```

证据优先级为：本组织历史、当前趋势、同类匿名基准、通用管理先验、经典变化类比。

## 8. 目标聚焦

目标卦不让用户直接从64卦中选择，而通过四个自然问题聚焦：

1. 三个月后，最不愿看到什么继续发生？
2. 如果短期只能守住一样，最想守住什么？
3. 出现什么真实变化时，会认为事情正在变好？
4. 为达到目标，什么代价不愿承担？

系统形成候选目标后，以管理语言请用户确认：

```text
你的目标不是立即恢复高速增长，而是：

• 稳定核心团队；
• 恢复可预测交付；
• 保留下一轮增长能力；
• 不继续透支管理者。

理解准确吗？
```

目标由系统帮助聚焦，但必须由用户最终确认和授权。

## 9. 一步一爻的路径优化

如果现状卦和目标卦之间有 `k` 爻不同，最短需要 `k` 步，并存在 `k!` 种可能顺序。系统选择综合代价最低的路径：

```math
\pi^*=\arg\min_\pi\sum_t\left[G(B_t,a_t)+\lambda C_t+\mu R_t+\nu U_t\right]
```

其中：

- `G`：预期自由能；
- `C_t`：行动成本；
- `R_t`：不可逆风险；
- `U_t`：组织承压。

路径优先选择：能验证关键假设、降低最大不确定性、带动其他部分改善、可逆且低成本、用户当前做得到的行动。

### 9.1 三种爻状态

| 状态 | 管理含义 |
| --- | --- |
| 未动 | 尚未开始 |
| 动爻 | 已经行动，等待结果验证 |
| 已变 | 验证条件成立，状态真正改变 |

采用执行状态变量：

```math
z_i\in\{\text{未启动},\text{验证中},\text{已确认},\text{未成立}\}
```

系统不能把“做了动作”等同于“实现了变化”。

## 10. 变爻动态仿真

每一步仿真分为五个阶段：

1. **解释选择**：为什么先变这一爻；
2. **呈现行动**：现实中具体由谁做什么；
3. **影响传播**：对其他爻位的改善、承压和不确定影响；
4. **路径重算**：根据变化重新判断下一步；
5. **进入验证**：明确最早信号、有效验证期、成功与风险条件。

示例：

```text
建议先变四爻 · 组织机制

当前最大阻点不是执行能力，
而是责任与权限没有对应。

这一步只做：
为A项目指定唯一交付负责人，
并同步移交最终审批权限。
```

验证卡：

```text
观察：审批等待时间
最早信号：3天
有效验证：2周
成功条件：连续两周低于24小时
风险信号：责任已移交，但权限仍被上级收回
```

只有验证通过，爻才稳定为新状态。

## 11. 动态仿真的指标

前台默认只突出三个主指标：

### 11.1 目标趋近度

回答：这一步是否让我们更接近用户确认的目标？

### 11.2 判断清晰度

回答：这一步是否帮助我们更清楚地判断真正原因？

```math
IG(a_t)=H[Q(s_t)]-\mathbb E H[Q(s_{t+1}\mid o_{t+1})]
```

### 11.3 组织承压度

综合资源、执行、关系、结构风险和不可逆性，回答：方向虽然合理，组织现在是否承受得住？

第二层按需展示结构杠杆度、可逆程度、见效窗口、行动成本、惯性偏转和后续选择开放度。

## 12. 经典与现代管理的知识桥梁

每条卦辞、爻辞、传文和林辞建立四层知识卡：

1. **原典层**：原文、版本、异文、卦位、变卦关系和主要意象；
2. **易理层**：时、位、中、正、应、承乘、比、进退、聚散、得失；
3. **管理层**：战略、组织、流程、人才、财务、市场、创新、风险和文化；
4. **行动层**：现代理论、诊断方法、策略、动作、观察信号和适用边界。

示例：

```text
经典状态
井——价值仍在，取用受阻

现代解释
组织并非缺少能力，而是价值流存在瓶颈。

对接理论
约束理论、核心能力、流程管理

当前策略
不急于增加资源，先疏通取用机制

一步行动
未来两周只保留一个最终审批人

验证信号
平均审批等待时间下降30%
```

## 13. 统一结果表达

每次正式输出采用五段式：

1. 林下判断；
2. 事实依据；
3. 易理启示；
4. 现代管理对接；
5. 当前一步。

所有结果明确标注：

```text
【事实】
【模型推断】
【经典启示】
【管理建议】
```

## 14. “点击即得”的纵深呈现

默认界面保持简洁，用户点击任意概念后逐层深入：

1. 一句话解释；
2. 用户当下情境；
3. 结构后果；
4. 易理与现代管理连接；
5. 验证方法和适用边界。

可点击对象包括概念、判断、指标、经典依据、管理方法、行动建议和路径节点。用户可以停在任何一层，不影响主流程。

稳定知识保证准确，用户情境保证相关：

```text
解释内容 = 稳定知识 + 用户情境实例
```

## 15. 情境化与个性化

系统维护六类经用户授权的必要情境：

1. 用户情境：角色、权限、经验、表达习惯与解释深度；
2. 组织情境：行业、规模、阶段、模式、结构与资源约束；
3. 问题情境：事件、事实、假设、时间压力与影响范围；
4. 目标情境：想实现、守住和不愿承担的内容；
5. 路径情境：已采取行动、当前变爻与被验证或否定的假设；
6. 承受情境：当前精力、压力，以及此刻需要分析、行动还是先稳定节奏。

每次回答前检查：我在对谁说、他正在处理哪件事、此刻最需要什么、建议与此前路径是否一致。

## 16. 进程陪护

每个议题建立一条同行线：

```text
事情发生
→ 初步理解
→ 形成现状
→ 聚焦目标
→ 选择变爻
→ 执行观察
→ 验证结果
→ 更新状态
→ 进入下一步
```

每一步形成陪护卡，包含当前行动、原因、负责人、观察信号、验证期限、成功条件和风险信号。

陪护只在行动前、最早信号出现、验证期到达、指标明显偏离、关键前提改变或用户主动回来时出现。用户可以随时继续、暂放、换路、修改目标或结束议题。

## 17. 暖心陪伴机制

每次表达遵循：

```text
陪伴表达 = 情境记忆 + 理性判断 + 情绪承接 + 轻量下一步
```

### 17.1 早安问候

无异常时：

> 早上好。今天没有需要立即处理的变化，A项目仍在观察期，我们周五再一起回看。

有关键行动时：

> 早上好。今天最值得关注的只有一件事：确认A项目的审批权限是否完成移交，其他事情暂时不用提前担心。

### 17.2 环境变化提醒

> 一个前提发生了变化：新增预算已经获批。此前“优先缩小范围”的建议需要重新评估。先不用执行原定动作，我已将路径标记为待更新。

### 17.3 善意宽慰

> 这一步没有产生预期改善，确实令人失望。但它排除了“审批人过多”这个主要假设，问题比两周前更清楚了。

### 17.4 克制鼓励

> 这是一个真实的小进展：团队第一次在没有上级介入的情况下完成了跨部门确认。

### 17.5 安静陪伴

没有事情发生时不制造提醒：

> 今天没有需要立即处理的变化。保持观察即可。

## 18. 陪伴强度与边界

用户可以选择安静、适度或同行三种陪伴强度，并设置工作时间、免打扰时间、周末提醒、允许主动推送的议题和通知渠道。

系统禁止：

- 用焦虑提高打开率；
- 每天机械问候；
- 无意义催促；
- 未经允许记录敏感情绪；
- 使用诱导依赖的表达；
- 以温暖措辞掩盖不确定性。

如果一条消息不能增加理解、减少负担或帮助行动，就不应推送。

## 19. 对标分析

对标遵循“与谁相比、为什么比较、哪些差距可改变”。

对标对象包括：

1. 组织自身历史；
2. 内部不同项目、团队和区域；
3. 行业、规模、阶段和模式相近的匿名组织；
4. 用户确认的阶段目标与最终目标。

对标不能只显示排名，必须解释比较条件、不可比因素、可改变部分，并转化为当前行动。

## 20. 复盘进化

极简复盘入口：

```text
结果与预期相比：

○ 更好
○ 基本符合
○ 没有改善
○ 出现新问题
```

复盘分为事实、判断、行动、机制和模型五层，沉淀为包含情境、当时判断、行动、结果、偏差、经验、不适用条件和下次提醒的组织经验卡。

系统不使用虚假的能力分数，而是呈现用户真实形成的管理方式变化。

## 21. 林下全景导航

“林下全景”是统一导航层，组织完整决策进程：

```text
事情缘起
→ 现状判断
→ 惯性方向
→ 目标
→ 路径
→ 当前变爻
→ 现实行动
→ 验证信号
→ 复盘结果
→ 更新状态
```

### 21.1 三个观察尺度

- 全局尺度：查看所有正在推进的目标和议题；
- 议题尺度：查看某件事从现状到目标的完整路径；
- 节点尺度：查看某一步为什么做、如何做、如何验证。

### 21.2 林下罗盘

任何页面保留三个入口：

```text
◎ 回到全景
→ 继续当前一步
⌕ 找任何内容
```

### 21.3 全局搜索与命令

用户可以直接输入“继续昨天的复盘”“查看所有未完成行动”“为什么先调整授权”“对比A项目和B项目”“如果先变五爻会怎样”等自然语言，系统直接到达相应目标。

### 21.4 一键回到当前任务

全系统始终提供“我现在该做什么”。如果正在等待，系统明确告诉用户无需增加动作。

全景不是一次展示全部复杂内容，而是保证任何内容随时可达。

## 22. 极简动态入口

首屏只保留：

```text
               [林下品牌标识]

           此刻，什么事最难决定？

    [用一句话说说，也可以只输入几个字]

               [陪我理一理]

          知道管理 · 林下系列
```

不设置注册墙，不要求先选择行业、卦象或管理模型。

背景由六条极细水平线构成抽象卦象：较长时间内只发生一次单爻细微变化，中心轴和品牌标识始终稳定；用户聚焦输入框后，动画安静下来。动效不循环闪烁，并支持减少动态效果。

## 23. 渐进式学习

系统可以复杂，但用户每一刻面对的必须简单。

### 23.1 双层产品

- 行动层默认显示发生了什么、林下如何理解、当前做什么、何时回来验证；
- 理解层按需展开证据、卦象、经典解释、替代路径和模型方法。

### 23.2 不设置传统新手教程

第一次使用只完成一个真实闭环：

```text
说一件事
→ 回答一个问题
→ 看见一个判断
→ 接受一个行动
→ 约定一次回看
```

### 23.3 功能按需要出现

- 第一次使用：输入、判断、行动、保存；
- 第一次回来：陪护、验证、状态更新；
- 完成第一次变爻：之卦、路径、动态仿真；
- 同时拥有多个议题：全景与优先级；
- 累积足够历史：自我对标、组织模式与长期进化。

### 23.4 一次只教一个概念

主界面采用用户语言，专业术语后置：

| 默认表达 | 深层概念 |
| --- | --- |
| 此刻 | 现状卦 |
| 保持现状会怎样 | 惯性卦 |
| 想走向哪里 | 目标卦 |
| 当前一步 | 变爻 |
| 更清楚了吗 | 信息增益 |
| 承受得住吗 | 承压度 |

系统根据用户熟悉度逐渐缩短解释，同时允许用户随时选择“简单一点”“展开专业依据”或“以后保持这种深度”。任何用户都可以永久停留在简单层。

## 24. 信息架构

首期只保留三个一级入口：

```text
此刻
├── 一句话输入
├── 当前判断
└── 关键不确定性

路径
├── 现状卦
├── 惯性卦
├── 目标卦
├── 变爻仿真
└── 当前一步

同行
├── 行动记录
├── 验证结果

├── 每周回看
└── 长期进化
```

移动端底部导航只保留“全景、当前一步、问林下”。

## 25. 可信与权威机制

权威来自可验证性，而不是复杂术语和绝对语气。

### 25.1 可追溯

每项判断均可查看事实、模型推断、对应卦爻、经典出处、现代转译和行动理由。

### 25.2 有置信度

```text
主要判断　　　　　　　61%
替代解释　　　　　　　27%
信息不足　　　　　　　12%
```

### 25.3 可纠正

用户可以修改事实、假设、目标和权重，系统随即更新。

### 25.4 可校准

持续记录当时判断、实际结果、偏差原因和模型调整。

### 25.5 有边界

系统明确表达证据不足、无法区分的解释、不适合下结论的情况，以及需要法律、财务、医疗或心理专业人士介入的事项。

## 26. 隐私与用户控制

用户可以查看“林下目前记得的内容”，并决定其仅用于本次对话、当前议题、长期组织记忆，或直接删除。

系统引用历史时说明来源。敏感信息遵循明确授权、最小采集、用途透明、随时可编辑和删除、不用于操纵用户等原则。

## 27. 知识审校

建立三级审校：

1. 文献审校：原文、版本、异文和卦变关系；
2. 易学审校：区分原典含义、历代解释和产品现代转译；
3. 管理审校：检查理论映射、行动可行性、行业限制和误导风险。

每张知识卡记录原典可信度、解释一致度、管理适用度、案例验证数和最近审校时间。

## 28. MVP实施路径

### 第一阶段：验证核心闭环

聚焦战略取舍、团队协同、项目交付、核心人才、现金增长和组织变革，完成极简入口、情境化追问、现状判断、目标澄清、一步变一爻、行动验证、基础陪护和进程保存。

### 第二阶段：经典桥接与动态路径

完成高频卦象和变化情境、经典知识卡、现代管理方法映射、变爻仿真、点击即得和替代路径比较。

### 第三阶段：个性化与全景

完成林下全景、多议题关联、自我对标、组织历史模型、个性化提醒和复盘进化。

### 第四阶段：完整易林网络

逐步完成4096种变化情境的文献校勘、易理解释、管理映射、行动模型、案例验证和适用边界。

## 29. 核心衡量指标

| 指标 | 衡量内容 |
| --- | --- |
| 首次有效判断率 | 用户是否认可系统理解了问题 |
| 首次闭环时长 | 从输入到获得行动需要多久 |
| 单次输入负担 | 用户需要提供多少信息 |
| 当前行动识别率 | 用户是否知道下一步做什么 |
| 首步接受率 | 用户是否愿意执行建议 |
| 验证完成率 | 用户是否回来记录真实结果 |
| 判断校准度 | 概率判断与结果是否匹配 |
| 路径调整合理度 | 新证据出现后能否正确改路 |
| 中断恢复时间 | 是否能在10秒内接上此前进程 |
| 导航到达效率 | 任意目标能否两步内触达 |
| 推送有效率 | 提醒是否真正有帮助 |
| 打扰关闭率 | 陪伴是否造成负担 |
| 独立判断提升 | 用户是否形成更清晰的管理方法 |
| 长期陪伴留存 | 是否形成持续复盘习惯 |

## 30. 最终产品闭环

```text
一句话进入
→ 林下理解当下情境
→ 形成现状判断
→ 推演惯性方向
→ 聚焦目标
→ 选择一步变爻
→ 动态仿真
→ 现实执行
→ 温和陪护
→ 情境对标
→ 结果复盘
→ 模型与用户共同进化
```

核心方法：

```text
现实证据
+ 主动推理
+ 周易状态智慧
+ 易林变化情境
+ 现代管理方法
+ 一步一爻
+ 进程陪护
+ 持续验证
```

《周易》帮助用户理解身处何局，《焦氏易林》帮助用户理解局势怎样变化，主动推理帮助用户判断哪条路径更可信，现代管理帮助用户把眼前这一步真正做好；林下则记得这一路为什么出发，并陪用户一步一步走到目标。
